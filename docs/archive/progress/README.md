# Progress Archive

Rolled-off history from the root [`PROGRESS.md`](../../../PROGRESS.md). The
active file holds only **current** state; completed entries move here so the
active State subsystem stays small and scannable, without losing the trail.

## Convention

- **One file per period:** `docs/archive/progress/<YYYY-QN>.md`
  (e.g. `2026-Q2.md`), or `<YYYY-MM>.md` for high-velocity projects.
- **Append-only:** archive files are a historical record; don't rewrite them.
- **Source map:** every archive file is linked from the `## Archive` section of
  `PROGRESS.md`. That link list is the index — if it isn't linked there, it's
  effectively lost (repository-as-source-of-truth).

## When & how to roll off

Owned by `docs/VERIFICATION.md` → "Archiving Growing State Files" (threshold,
steps, source-map rule). This file only holds the archived periods.
